## Purpose of This Document

Raw competitor intelligence for building the Klai Docs product page. Klai Docs is an AI-powered document editor where every edit automatically feeds back into the organisation's knowledge base in real time. It is the "editable exposure layer" of the Klai Knowledge system: the permanent fix for the stale wiki problem, always included with Klai Knowledge, EU-hosted and private.

## Cross-Competitor Analysis

### What All Three Have in Common

| Pattern | Notion | Coda | Confluence |
|---|---|---|---|
| AI as creation assistant | Yes (AI blocks) | Yes (Coda AI) | Yes (Rovo) |
| AI as search/retrieval | Yes (Enterprise Search) | Partial | Yes (Rovo Search) |
| Real-time collaboration | Yes | Yes | Yes (Live docs) |
| Templates as on-ramp | Yes | Yes | Yes |
| Single source of truth claim | Yes | Yes | Yes |
| Docs-to-KB feedback loop | **No** | **No** | **No** |
| EU hosting / privacy | No | No | No |

### The Gap None of Them Fill

All three describe knowledge as something you build and then search. None of them frame the act of writing a document as an automatic update to the organisation's knowledge base. The feedback loop is always manual or implicit:

- Notion: you centralise docs, AI searches them. Freshness is your responsibility.
- Coda: edits sync across tables. Knowledge is queried, not maintained.
- Confluence: "live docs" for real-time editing; Rovo summarises pages. Freshness is addressed reactively.

**Klai Docs' structural differentiator:** every edit immediately and automatically updates the knowledge base. There is no "publish to knowledge base" step. The editor is the knowledge base interface. This is architecturally distinct from all three competitors.

### How Competitors Frame "Always Up to Date"
- Notion: auto-updating table of contents; live Figma embeds; real-time co-editing
- Coda: "edits sync everywhere"; single source of truth
- Confluence: "live docs"; AI-powered summaries; audio briefings

None of these address the root cause of stale knowledge: the separation between the writing layer and the knowledge layer. They add freshness features on top of a fundamentally static architecture.

### Pain Points Competitors Acknowledge (Klai Should Address Directly)
1. Stale documentation / outdated wikis ("old docs" — Confluence calls this out directly)
2. Information silos across tools
3. Time wasted finding current information
4. The blank page problem
5. Async collaboration across time zones
6. Tool fragmentation (Coda leads on this)
7. Enterprise governance and compliance (Notion leads on this)

### Pain Points No Competitor Addresses (Klai Opportunity)
1. Data sovereignty — none mention EU hosting or GDPR-first architecture
2. The root cause of stale wikis — the structural separation between writing and the knowledge base
3. Knowledge that degrades the moment a doc is "published" — publishing as a moment of decay onset
4. AI trained on your own private knowledge, not a shared US-hosted model

### Copy Angles Worth Borrowing / Contrasting

| Competitor line | Klai contrast angle |
|---|---|
| Notion: "Centralize all your knowledge in Notion instead" | Klai: Knowledge is centralised automatically — you just write |
| Coda: "Tables talk to each other, edits sync everywhere" | Klai: Your docs talk to your knowledge base. Every edit, automatically |
| Confluence: "Ditch your old docs" | Klai: Ditch the stale wiki. Not by migrating to a new tool — by ending the architecture that makes wikis go stale |
| Notion AI: "Meet your 24/7 AI team" | Klai: An AI that actually knows what your organisation knows today, not last quarter |
| Confluence: "One place for all your ideas, docs, knowledge" | Klai: One act — writing a doc — does all three at once |

### Pricing / Business Model Signals
- Notion: freemium, per-seat scaling, AI as paid add-on
- Coda: "pay for makers, not contributors" (unique model, reduces per-seat cost)

## Klai Docs: what's actually built (code audit 2026-03-24)

**Technology:** Gitea-backed Next.js document editor. `klai-docs/lib/gitea.ts` handles document storage. Private repos per organisation. Do NOT mention Gitea, Next.js, or "Gitea-backed" in user copy.

**Knowledge sync:** Every edit in Klai Docs feeds into Klai Knowledge (Qdrant) automatically. This is the core architectural differentiator vs. Notion, Coda, Confluence.

**Collaboration:** Standard document editor features. Real-time co-editing planned/partial.

**Always included with Knowledge:** Docs is the editing interface for the knowledge base. One product, not two.

**Copy framework (validated):**
- Lead with: "Write it once. Your whole team finds it. Forever."
- Then: every edit updates the knowledge base automatically (no publish step, no migration)
- Then: stale wiki problem solved at architecture level, not with a freshness badge
- Privacy as unlock: EU-hosted, open source, DPA included
- NOT: "Gitea-backed", "open source foundation at the storage layer", any tech names

## Klai Docs Positioning Summary (Derived from Research)

**The problem none of them solve:** The wiki goes stale because writing and knowledge maintenance are two separate acts. Every competitor treats the document as the artifact and the knowledge base as the archive. Klai Docs treats them as the same thing.

**Unique angles for the Klai Docs product page:**

1. **The feedback loop angle** — "Every doc you write instantly updates the knowledge base. No publish step. No migration. No maintenance."
2. **The stale wiki angle** — "The wiki is stale because it was always separate from where people actually write. Klai Docs closes that gap permanently."
3. **The included angle** — "Always included with Klai Knowledge. One system, not two products."
4. **The EU angle** — "Your knowledge stays in Europe. Under your control. Always."
5. **The AI context angle** — "Your AI answers from what your team wrote this morning, not from a training snapshot from last year."

**What to borrow from competitors:**
- Notion's role-based framing (PMs, designers, engineers) — concrete and scannable
- Coda's "single source of truth" language — widely understood pain point
- Confluence's direct call-out of "old docs" — permission to be blunt about legacy tools
- Notion AI's "full workspace context" framing — but claim it as real-time, not just indexed
