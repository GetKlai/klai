# Competitor Research: Klai Knowledge

**Purpose:** Input for building the Klai Knowledge product page — an organisational knowledge base with RAG, automatic document feedback loop, and EU-private hosting.

**Researched:** 2026-03-24
**Sources scraped:** notion.com/product/ai, notion.com (homepage), notion.com/use-case/knowledge-base, glean.com, glean.com/product/assistant, glean.com/solutions/knowledge-management, getguru.com

## Klai Knowledge differentiators to keep in mind

- **Automatic feedback loop:** documents created in Klai Docs feed back into Knowledge automatically — no manual uploads, no stale bases
- **Chat integration:** Knowledge feeds context directly into every Klai Chat session
- **Access controls per user:** permission-aware retrieval
- **EU-hosted, private:** data never leaves EU infrastructure; no US cloud exposure
- **Fully open-source stack**

## Cross-competitor analysis

### Terminology comparison

| Concept | Notion | Glean | Guru |
|---|---|---|---|
| The knowledge store | "knowledge base", "wiki" | "enterprise graph", "company knowledge" | "source of truth", "verified company knowledge" |
| AI over own data | "context from your workspace" | "system of context", "company context" | "AI insights grounded in your knowledge" |
| Keeping knowledge fresh | "verified badge" (manual) | "verify and deprecate" (manual governance) | "automate the upkeep" (automated) |
| Attribution/citations | Not prominent | Not prominent | "Always cited" (core claim) |
| Data privacy framing | Zero data retention, no training | Permissions-enforced retrieval | Permission-aware governance |
| "Memory" language | None | None | None (closest: "source of truth") |
| Pricing model | Freemium + usage credits | Demo-gated, ROI framing | Tiered; Starter/Builder/Expert |

### What no competitor says

None of the three use any of these framings:
- "Your knowledge base that writes itself"
- Automatic document-to-knowledge feedback loop
- Docs that become knowledge automatically
- Knowledge feeding into every conversation automatically
- EU-hosted, data sovereign knowledge
- Open-source knowledge infrastructure
- Privacy-first AI memory

### Shared patterns worth noting

1. **None explain RAG technically** — all use plain-language "your data", "company knowledge", "context"
2. **"Source of truth" is crowded** — Guru leads with it; Notion implies it; worth differentiating
3. **Freshness/accuracy is a pain point** — all three address it, but differently (manual badges, governance tools, automated upkeep)
4. **Permissions are table stakes** — all three mention access controls; not a differentiator on its own
5. **Citations/attribution is Guru's niche** — "always cited" is unique to Guru among these three
6. **Integration breadth is Glean's niche** — 100+ connectors is their strongest claim
7. **Workflow consolidation is Notion's niche** — replacing multiple tools in one platform

### Positioning opportunity for Klai Knowledge

| Klai differentiator | Gap in competitor landscape |
|---|---|
| Docs auto-feed Knowledge (feedback loop) | No competitor offers automatic, seamless doc-to-knowledge pipeline |
| Knowledge feeds Chat context | Glean/Notion partially do this, but it requires manual setup or agent configuration |
| EU-hosted, data sovereign | None of the three make EU hosting or data sovereignty a headline claim |
| Open-source stack | None |
| No US cloud exposure | None explicitly claim this |
| Integrated with all Klai products (Chat, Docs, Portal) | Competitors integrate via connectors; Klai's is native |

## Klai Knowledge: what's actually built (code audit 2026-03-24)

**Storage:** Qdrant `klai_knowledge` collection in `klai-portal/backend/app/api/knowledge.py`. Supports org scope and personal scope.

**Automatic document ingestion:** Klai Docs edits feed Knowledge automatically. Scribe transcripts can be saved to Knowledge. `klai-connector` (GitHub adapter in `klai-connector/app/adapters/github.py`) syncs .md, .txt, .pdf, .docx, .rst, .html, .csv files.

**Chat integration:** Chat knowledge base context is near-complete (described as "bijna klaar" by founder 2026-03-24). Sell as live.

**Access controls:** Per-user, permission-aware retrieval. Already implemented.

**Copy framework (validated):**
- Lead with scenario: "A new colleague has a question. They ask Klai. They get the answer. Without interrupting you."
- Headline: "A knowledge base that writes itself." (confirmed strong)
- Not: feature list in the subheadline (no "no manual uploads, no stale wikis" as the opener)
- Key differentiator vs Notion/Glean/Guru: the automatic doc-to-knowledge feedback loop. None of them have this.
- Privacy as unlock: EU-hosted, access-controlled, open source stack

### Recommended framing directions for Klai Knowledge copy

- Lead with the feedback loop as the category-defining claim: knowledge that grows automatically
- "Your knowledge base writes itself" or "Every document becomes knowledge" as a possible headline direction
- EU and privacy as trust layer, not headline (table stakes for EU audience but differentiating vs. US competitors)
- Avoid "source of truth" (Guru owns it); instead consider "living knowledge" or "knowledge that keeps up"
- Citations/attribution is worth including (matches Guru's "always cited") — Klai Knowledge should surface sources in Chat
- Pricing: competitors either hide pricing (Glean) or use freemium (Notion, Guru) — Klai can differentiate with transparent EU pricing
